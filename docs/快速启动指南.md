## 快速开始

### 前置要求

- Docker和Docker Compose已安装
- ngrok账号（免费注册：https://ngrok.com/signup）

### 一键启动

#### Linux/Mac系统

```bash
cd /path/to/Travel_agent
bash start.sh
```

#### Windows系统

```cmd
cd D:\JavaPro\Travel_agent
start.bat
```

脚本会自动：
1. 检查Docker环境
2. 配置ngrok authtoken
3. 构建并启动所有服务（包括ngrok）
4. 显示公网访问地址

### 手动启动

#### 1. 配置环境变量

创建或编辑 `.env` 文件：

```bash
# Ngrok配置
NGROK_AUTHTOKEN=2abc...xyz  # 你的ngrok authtoken

# 其他环境变量...
```

#### 2. 启动服务

```bash
docker-compose up -d --build
```

#### 3. 查看ngrok公网地址

```bash
docker-compose logs -f ngrok
```

或访问：`http://localhost:4040`

## Docker Compose集成说明

### 服务架构

项目使用统一的Docker网络 `travel-network`，所有服务都在同一个网络中：

```
┌─────────────────────────────────────────┐
│           Docker Network                │
│         (travel-network)                │
├─────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ Frontend │  │ Backend  │  │ Ngrok  │ │
│  │   :80    │  │  :8000   │  │ :4040  │ │
│  └──────────┘  └──────────┘  └────────┘ │
│                                         │
│  ┌──────────┐  ┌──────────┐  ┌───────┐ │
│  │  MySQL   │  │ MongoDB  │  │ Redis │ │
│  │  :3306   │  │  :27017  │  │ :6379 │ │
│  └──────────┘  └──────────┘  └───────┘ │
└─────────────────────────────────────────┘
```

### Ngrok服务配置

在 `docker-compose.yml` 中，ngrok服务配置如下：

```yaml
ngrok:
  image: ngrok/ngrok:latest
  command: start --all --config /etc/ngrok/ngrok.yml
  ports:
    - "4040:4040"  # Ngrok Web UI
  volumes:
    - ./backend/scripts/ngrok.yml:/etc/ngrok/ngrok.yml:ro
  environment:
    - NGROK_AUTHTOKEN=${NGROK_AUTHTOKEN}
  depends_on:
    - backend
    - frontend
  restart: always
  networks:
    - travel-network
```

### 网络配置优势

1. **容器间通信**: ngrok可以通过容器名称（`backend:8000`、`frontend:80`）访问其他服务
2. **自动启动**: ngrok会在backend和frontend启动后自动启动
3. **统一管理**: 所有服务通过 `docker-compose` 统一管理
4. **网络隔离**: 服务在独立的Docker网络中运行，更加安全

## Ngrok配置

### 配置文件位置

`backend/scripts/ngrok.yml`

### 配置说明

```yaml
version: "3"

tunnels:
  # 后端API隧道
  backend-api:
    proto: http
    addr: backend:8000  # 使用Docker容器名称
    domain: your-subdomain.ngrok-free.app  # 可选：自定义域名
    bind_tls: true
    inspect: false
    schemes:
      - https
      - http
    basic_auth:  # 可选：添加基础认证
      - "admin:your_password_here"
    options:
      timeout: 30s
      max_connections: 100

  # 前端隧道
  frontend:
    proto: http
    addr: frontend:80  # 使用Docker容器名称
    bind_tls: true
    inspect: false
    schemes:
      - https
      - http
    options:
      timeout: 30s
      max_connections: 100
```

### 获取Ngrok Authtoken

1. 访问 https://ngrok.com/signup 注册账号
2. 登录后访问 https://dashboard.ngrok.com/get-started/your-authtoken
3. 复制你的authtoken（格式：`2abc...xyz`）

### 配置Authtoken

#### 方式一：使用启动脚本（推荐）

运行 `start.sh` 或 `start.bat`，脚本会提示你输入authtoken并自动配置。

#### 方式二：手动配置

编辑 `.env` 文件：

```bash
NGROK_AUTHTOKEN=2abc...xyz
```

然后重启服务：

```bash
docker-compose restart ngrok
```

## 常用命令

### 服务管理

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启所有服务
docker-compose restart

# 重启特定服务
docker-compose restart backend
docker-compose restart ngrok

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f ngrok
```

### Ngrok管理

```bash
# 查看ngrok日志
docker-compose logs -f ngrok

# 访问ngrok Web UI
http://localhost:4040

# 重启ngrok
docker-compose restart ngrok

# 重新构建ngrok
docker-compose up -d --build ngrok
```

### 数据库管理

```bash
# 进入MySQL容器
docker-compose exec mysql bash

# 进入MongoDB容器
docker-compose exec mongodb bash

# 进入Redis容器
docker-compose exec redis bash
```

## 故障排查

### 1. Ngrok启动失败

**问题**: ngrok容器无法启动

**解决**:
```bash
# 查看ngrok日志
docker-compose logs ngrok

# 检查authtoken是否配置
docker-compose exec ngrok ngrok config check

# 重新配置authtoken
docker-compose exec ngrok ngrok config add-authtoken YOUR_TOKEN
```

### 2. 无法访问公网地址

**问题**: ngrok启动但无法访问公网地址

**解决**:
```bash
# 检查服务是否正常运行
docker-compose ps

# 检查backend和frontend是否启动
docker-compose logs backend
docker-compose logs frontend

# 检查网络连接
docker-compose exec ngrok ping backend
docker-compose exec ngrok ping frontend
```

### 3. Authtoken未配置

**问题**: `ERR_NGROK_4018` 错误

**解决**:
```bash
# 编辑.env文件，添加NGROK_AUTHTOKEN
echo "NGROK_AUTHTOKEN=your_token" >> .env

# 重启ngrok
docker-compose restart ngrok
```

### 4. 端口被占用

**问题**: 端口80、8000、4040等被占用

**解决**:
```bash
# 查看端口占用
netstat -tlnp | grep :80
netstat -tlnp | grep :8000
netstat -tlnp | grep :4040

# 停止占用端口的进程
kill -9 <PID>

# 或修改docker-compose.yml中的端口映射
```

### 5. 容器间无法通信

**问题**: ngrok无法访问backend或frontend

**解决**:
```bash
# 检查网络
docker network ls
docker network inspect travel-agent_travel-network

# 检查容器是否在同一网络
docker-compose exec ngrok ping backend
docker-compose exec ngrok ping frontend

# 重新创建网络
docker-compose down
docker-compose up -d
```

### 6. 免费域名经常变化

**问题**: 每次重启ngrok，域名都会变化

**解决**:
- 免费账户的域名会随机变化，这是正常现象
- 如需固定域名，需要升级到付费计划
- 或使用其他内网穿透工具（如frp自建）

## 高级配置

### 自定义域名（付费）

1. 升级ngrok账户
2. 在dashboard中添加域名
3. 修改 `backend/scripts/ngrok.yml`:

```yaml
tunnels:
  backend-api:
    domain: api.yourdomain.com
    # ...
```

### 基础认证保护

编辑 `backend/scripts/ngrok.yml`:

```yaml
tunnels:
  backend-api:
    basic_auth:
      - "admin:secure_password"
    # ...
```

### 请求检查和调试

编辑 `backend/scripts/ngrok.yml`:

```yaml
tunnels:
  backend-api:
    inspect: true  # 启用请求检查
    # ...
```

然后访问 `http://localhost:4040` 查看请求详情。

### 添加更多隧道

编辑 `backend/scripts/ngrok.yml`，添加新的隧道配置：

```yaml
tunnels:
  backend-api:
    # ...

  frontend:
    # ...

  # 添加新的隧道
  admin-panel:
    proto: http
    addr: admin:8080
    bind_tls: true
    # ...
```

## 安全建议

1. **不要暴露敏感端口**: 不要将MySQL、MongoDB、Redis等数据库端口通过ngrok暴露到公网
2. **使用基础认证**: 为重要的隧道添加密码保护
3. **定期更新**: 保持ngrok版本最新
4. **监控访问日志**: 定期检查ngrok Web UI的访问日志
5. **生产环境慎用**: 免费版ngrok不适合生产环境，建议使用固定域名或自建frp
